## `> architecture_principles`

```
Clean Architecture     →  Separación clara de responsabilidades, capas que no se contaminan
Domain-Driven Design   →  El negocio primero, la tecnología después
SOLID Principles       →  Código que escala sin romperse
Event-Driven           →  Sistemas desacoplados que responden al cambio en tiempo real
CI/CD First            →  Si no se puede desplegar en minutos, hay un problema
Security by Design     →  Autenticación, roles y auditoría desde el día cero
```
